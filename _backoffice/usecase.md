# 백오피스 유스케이스

## 1. 관리자 인증 (Authentication)

- **인증 방식**: ID/Password 기반의 폼 로그인을 사용합니다.
- **세션 관리**:
    - 인증에 성공하면, 서버는 `HttpSession`에 로그인한 관리자의 계정 ID(`accountId`)를 `"ACCOUNT_ID"`라는 키로 저장합니다.
    - 이후 모든 요청은 이 세션 정보를 통해 인증 상태를 확인합니다.
    - 별도의 JWT 토큰은 사용하지 않습니다.
    - **세션 타임아웃**: 보안을 위해, 사용자의 마지막 활동으로부터 **60분**이 지나면 세션은 서버에서 자동으로 파기됩니다.
- **API 엔드포인트**:
    - `POST /api/auth/login`: 로그인을 시도합니다.
        - 성공 시: HTTP 200 OK와 함께 세션이 생성됩니다.
        - 실패 (ID/PW 불일치) 시: HTTP 401 Unauthorized를 반환합니다.
        - 실패 (계정 잠김) 시: HTTP 403 Forbidden을 반환합니다.
    - `POST /api/auth/logout`: 로그아웃을 수행합니다.
        - 성공 시: 현재 세션을 무효화(invalidate)하고 HTTP 200 OK를 반환합니다.

## 2. 전역 예외 처리 (Global Exception Handling)

- `@RestControllerAdvice`를 사용하여 애플리케이션 전반의 예외를 일관되게 처리합니다.
- 클라이언트에는 에러의 원인과 관련된 HTTP 상태 코드, 그리고 한글로 된 에러 메시지를 JSON 형식(`{"message": "..."}`)으로 반환합니다.
- 예상치 못한 서버 내부 오류(500) 발생 시, 서버에만 상세 로그를 기록하고 클라이언트에는 "서버 내부 오류"라는 일반적인 메시지를 전달합니다.
