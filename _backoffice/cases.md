# 백오피스 도메인 규칙

## 1. 관리자 계정 (AccountBackoffice)

- **목적**: 백오피스 시스템에 접근할 수 있는 관리자의 '인증' 정보를 관리합니다.
- **플랫폼 연관 관계**:
    - 모든 관리자 계정은 `_platform` 서비스의 `users` 테이블에 존재하는 특정 사용자와 반드시 1:1로 연결됩니다.
    - 이 연결은 `userId` 필드를 통해 이루어지며, 이를 통해 관리자의 실제 프로필 정보(이름, 전화번호 등)를 `_platform`에서 조회할 수 있습니다.
- **로그인 실패 정책**:
    - 로그인 실패 시 `failed_login_attempts` 카운트가 1씩 증가합니다.
    - 로그인 성공 시, 이 카운트는 0으로 초기화됩니다.
- **계정 잠금 정책**:
    - `failed_login_attempts`가 5회 이상이 되면, 해당 계정은 `is_locked` 상태가 `true`로 변경되어 잠금 처리됩니다.
    - 계정이 잠긴 후에는 올바른 비밀번호를 입력해도 로그인이 불가능합니다.

## 2. 관리자 로그인 이력 (AdminLoginHistory)

- **목적**: 관리자 계정의 모든 로그인 '시도'를 추적하고 감사(Audit)하기 위해 기록합니다.
- **기록 시점**: 로그인 성공, 실패(비밀번호 오류, 계정 잠김 등) 등 결과와 관계없이 모든 로그인 시도 시에 기록이 생성됩니다.
- **기록 정보**:
    - `accountId`: 어떤 관리자 계정이 시도했는지 식별합니다.
    - `isSuccess`: 로그인 시도의 성공/실패 여부를 나타냅니다.
    - `ipAddress`: 어디서 접속을 시도했는지 추적합니다.
    - `userAgent`: 어떤 환경/도구(브라우저, 앱, 스크립트 등)로 접속을 시도했는지 추적합니다.
- **트랜잭션 독립성**: 로그인 로직의 주된 트랜잭션이 롤백되더라도, 로그인 이력 기록은 별도의 트랜잭션으로 처리되어 반드시 데이터베이스에 저장됩니다.
